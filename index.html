IST 615 · Cloud Management · Winter 2026

Real-Time Patient Readmission Risk Monitoring on AWS

Designed a complete production cloud architecture for a hospital system to predict and intervene on 30-day patient readmissions in real time. The system ingests clinical data through API Gateway, streams it through Kinesis, scores patients with a SageMaker ML model, and triggers care workflows through EventBridge. Dashboards in QuickSight serve two audiences: care managers and hospital executives.

Financial model showed 71.5% first-year ROI — 12 prevented readmissions generating $212,400 in savings against $123,840 in AWS operating costs. Architecture designed to be fully HIPAA-compliant: VPC isolation, TLS, KMS encryption, least-privilege IAM, CloudTrail audit logging.

Amazon Web Services Kinesis SageMaker Lambda DynamoDB QuickSight HIPAA
IST 707 · Applied Machine Learning · Spring 2025

Predictive Risk Scoring for Mortgage Borrowers

Built a mortgage risk classification model using the relational database designed in IST 656 the prior semester. Merged Borrower and Loan tables, engineered features from credit behavior and loan characteristics, and trained three models using 5-fold cross-validation.

F1 = 0.987 (mean, all three models). XGBoost selected for production. Deployed as a Streamlit web application where loan officers adjust risk thresholds and see approval rate impact in real time. The model is hidden; the decision is what the interface surfaces.

Python XGBoost scikit-learn pandas Streamlit 5-fold CV
IST 691 · Deep Learning in Practice · Spring 2025

Sentiment Classification in SEC Filings

Compared three NLP approaches on the Financial PhraseBank dataset — short sentences from SEC filings labeled positive, negative, or neutral. Built and evaluated a TF-IDF baseline, an LSTM with GloVe embeddings, and a fine-tuned FinBERT transformer.

Results: Baseline 90%, LSTM 72% (insufficient data for deep learning from scratch), FinBERT 96%. Key finding: on small, domain-specific datasets, transfer learning from a domain-relevant pretrained model wins. Also wrote a manual PyTorch training loop after Hugging Face Trainer had version conflicts — came away understanding transformer fine-tuning at the implementation level.

PyTorch FinBERT Hugging Face GloVe LSTM NLP
IST 656 · Database Management · Winter 2025

Mortgage Risk Relational Database (All Tricks LLC)

Designed and implemented a relational database for a fictional mortgage risk consulting firm. Three core entities — Borrower_Info, Loan_Info, Property_Info — with primary and foreign keys, referential integrity constraints, and normalization decisions made to support downstream risk analysis.

Wrote SQL scripts for schema creation, constraint enforcement, and analytical queries joining borrower credit profiles with loan terms and property data. The database became the data source for the IST 707 machine learning project the following semester — a two-semester pipeline that wasn't planned but emerged from working on real problems in sequence.

SQL PostgreSQL ERD Design Schema Design Referential Integrity

Background

About Me

I came into this program as a data analyst who was good at his job but honest about what he didn't know. I could clean a dataset, build a dashboard, and present findings to a room full of school leaders.

What I couldn't do was build the systems behind the data. That gap is what brought me to Syracuse.

By the time I submitted this portfolio, I had been promoted to Senior Data Analyst at Success Academy Charter Schools in New York City and was preparing to move into a Cloud Solutions Engineer role. Every course I took was oriented toward that goal: building data systems that work at scale, in the cloud, and in production.

I came in able to query data someone else built the system for. That is not where I am leaving it.
